# Classified Recurring Bills Output Process

This note documents the workbook generated by:

```bash
PYTHONPATH="./src" python scripts/export_all_bills_classified.py
```

Output:

```text
data/snapshots/All_Private_Bills_K1_K25_classified.xlsx
```

## Input Sources

- `KNS_Bill`: source bill identity, Knesset number, private bill number, title, and subtype.
- `KNS_DocumentBill`: source document URL. The scan prefers `הצעת חוק לדיון מוקדם` and falls back to `הצעת חוק לקריאה הראשונה`.
- `data/external/knesset_docs/`: local cache of downloaded `.doc`, `.docx`, and `.pdf` source documents.
- `bill_classifications_doc_full`: warehouse table containing the document-scan results used by the export.

## Document Parsing

The parser downloads or reuses the cached source document and extracts text with:

- `textutil` for `.doc` / `.docx`
- `pypdf` for `.pdf`

Only the source document URL identifies the source document. It must not be reused as target-bill evidence.

## Reference Extraction

The scan searches the explanatory text for explicit recurrence phrases such as:

- `הצעת חוק זהה`
- `הצעות חוק זהות`
- `הצעת חוק דומה`
- `הצעות חוק דומות`
- `הצעות חוק דומות בעיקרן`
- references to a proposal submitted or laid before an earlier Knesset

For each recurrence context, it extracts target evidence only when it appears in the text:

- target Knesset, from Hebrew or numeric Knesset wording
- target private bill number, from `פ/NNN` or `פ/NNN/KK`
- target URL, currently not extracted by this parser

If a source document contains several references, the workbook `Reference Resolution` sheet writes one row per reference with `reference_index = 1, 2, 3...`.

## Target Bill Resolution

Resolution is allowed only when target evidence is independently present in the reference text.

Allowed:

- explicit private bill number plus explicit target Knesset, for example `פ/3068/18`
- private bill number plus a target Knesset phrase in the same recurrence context, for example `הכנסת השמונה-עשרה ... פ/3068`

Not allowed:

- copying the source Knesset from `KNS_Bill.KnessetNum`
- reading the source Knesset from a URL path such as `/11/law/...`
- using the source document id as a target bill number
- resolving a reference-less phrase by matching the source bill title to another bill
- resolving a bare `פ/NNN` by trying the same or prior Knesset

When a phrase says an older Knesset but gives no bill number or link, the target remains unresolved. Example:

```text
https://fs.knesset.gov.il//11/law/11_lst_534232.doc
```

The source is Knesset 11. The text says an identical bill was submitted in Knesset 10, but gives no `פ/NNN` and no target URL. The workbook may set `target_knesset_extracted = 10`, but must leave `target_bill_number_extracted`, `resolved_target_bill_id`, and final target bill metadata blank with:

```text
target_resolution_status = unresolved_no_link_or_number
```

## Relation Classification

`explicit_relation_type` and `final_relation_type` are based only on explicit phrases in the document text.

The pipeline does not compare normalized legal amendment text, does not remove explanatory notes for legal-text comparison, and does not compute legal-text identity or similarity.

Current values:

- `identical`: explicit phrase evidence such as `זהה` / `זהות`
- `similar`: explicit phrase evidence such as `דומה` / `דומות`
- `unknown`: reference evidence exists but no explicit identical/similar phrase was classified
- blank: no recurrence phrase

## Confidence And Scores

The workbook does not contain manually validated accuracy percentages.

Confidence columns are heuristic extraction/resolution confidence values:

- `target_resolution_confidence`: clearer alias for the selected resolution confidence.
- `reference_resolution_confidence`: legacy column retained for compatibility.

Typical values:

- `0.99`: explicit private number and explicit target Knesset resolved to a warehouse bill.
- `0.92`: private number resolved with a target Knesset phrase in the same context.
- blank: unresolved reference.

These values are not accuracy, precision, recall, or validation-set performance.

## Workbook Sheets

- `Classified Bills`: one row per source bill, with source aliases, effective-original fields, resolution status, warnings, and legacy compatibility columns.
- `Reference Resolution`: one row per extracted reference or unresolved phrase evidence.
- `Data Dictionary`: every exported column, its worksheet, kind, definition, and possible values.

## Known Limitations

- Old Knesset documents often mention prior bills without a private bill number or link. Those rows remain unresolved by design.
- OCR or legacy `.doc` extraction can lose punctuation or layout.
- `target_url_extracted` is currently blank because target URL extraction is not implemented.
- Source document URLs prove only source-document identity, not target-bill identity.
- Private bill numbers are not globally unique across Knessets; a bare `פ/NNN` is not enough for target resolution.
- `explicit_relation_type` is phrase-based and must not be read as legal-text similarity.

## Validation

Regression tests cover:

- the professor example URL `https://fs.knesset.gov.il//11/law/11_lst_534232.doc`
- no source-Knesset fallback for bare `פ/NNN`
- no source-name fallback for reference-less older-Knesset phrases
- one row per reference in the `Reference Resolution` worksheet
- preservation of all resolved references in multi-reference documents
- data dictionary worksheet creation

Recommended command:

```bash
PYTHONPATH="./src" .venv/bin/python -m pytest \
  tests/test_recurring_bills_knesset_docs.py \
  tests/test_recurring_bills_export_resolution.py -q
```
